Getting Started with WebSphere Application Server sample readme

The getting started samples are provided for a quick introduction to eXtreme Scale functionality and basic operation in a WebSphere Application Server environment.

The samples consist of servlet based server-side application, including source, provided to start a simple grid. In addition, a client servlet application, including source, is provided to run simple create, read, update, and delete (CRUD) functions to this basic grid. The client application can also be used to run simple create, read, update and delete functions to a simple grid hosted on a WebSphere DataPower XC10 appliance.

Software requirements

  • WebSphere eXtreme Scale Version 8.5 or later
  • WebSphere Application Server Version 7 or later
  • Eclipse IDE for Java EE Developers, Indigo or later with the WebSphere Application Server Developer Tools or Rational Application Developer IDE.

Before installing the sample applications, you must either perform a full installation of WebSphere eXtreme Scale to use both the client and server sample applications or a WebSphere eXtreme Scale client only install to use just the client application. WebSphere eXtreme scale must be installed in the same directory which you installed your WebSphere Application Server. You must also perform Profile augmentation for the WebSphere Application Server profiles in order to support eXtreme Scale features.

Installing the samples in the WebSphere Application Server

Use the following basic steps to install the client and server applications into WebSphere Application Server.

  1. In a browser, log in to the WebSphere Integrated Solution Console.
  2. In the left-hand pane, expand Applications, and then click New Application.
  3. Click New Enterprise Application.
  4. Click Browse, navigate to the directory of the downloaded sample:
    1. To install the client, select GettingStartedWebClient.ear.
    2. Click Open.
    3. Click Next.
    4. Select Fast Path, and then click Next.
    5. Accept the default installation options by clicking Next.
    6. Select your application server or cluster and click Apply, then click Next.
    7. Accept the default virtual hosts for Web modules by clicking Next.
    8. Accept the default metadata for modules by clicking Next.
    9. Verify that the Summary is correct, and then click Finish.
    10. Click Save.
  5. To install the server component, select GettingStartedWebServer.ear, repeat steps 2 through 4.
  6. In the left-hand pane, expand Application Types, and select WebSphere enterprise applications.
  7. Select the application or applications that you installed in steps 2 through 4, and click Start.
  8. Verify that the application or applications have successfully started.
  9. Log out of the WebSphere Integrated Solution Console.

Running the sample

Running the server application on WebSphere Application Server

The servlet based server application performs no other function than being used as a vehicle to deploy a WAR file that contains objectGrid.xml, and objectGridDeployment.xml files. When this application is started, a grid container based on the configuration details in the previous .xml files will start and register with the configured catalog server, or an embedded catalog server if one is not configured.

You can verify the application is running using a browser.

  1. In a browser, enter the following server servlet URL: http://hostname:port/GettingStartedWebServer/ServerServlet

    The <hostname> is the host name or IP address of WebSphere Application Server and <port> is the web server port for your WebSphere Application Server. See the following server servlet example:

    http://localhost:9080/GettingStartedWebServer/ServerServlet

If the application is running, you will see the following message:

WebSphere eXtreme Scale Getting Started Sample
This sample application starts an ObjectGrid container.
The ObjectGrid container is active.

Running the client application on WebSphere Application Server

The client application servlet is a simple ObjectGrid client that connects to a remote or embedded ObjectGrid and performs Create, Read, Update and Delete (CRUD) operations. This client can be used with a WebSphere eXtreme Scale grid or a simple grid on a WebSphere DataPower XC10 appliance.

  1. In a browser, enter the following server servlet URL: http://hostname:port/GettingStartedWebClient/ClientServlet

    The <hostname> is the host name or IP address of WebSphere Application Server and <port> is the web server port for your WebSphere Application Server. See the following server servlet example:

    http://localhost:9080/GettingStartedWebClient/ClientServlet
  2. In the Client Connection section, leave this section blank to use the configured or embedded catalog server, or enter the hostname or IP address and port of the Remote catalog service endpoint in the form of hostname:port.
  3. After entering the catalog service endpoint, click Connect to test the connection. If the connection is successful, you will see true for Client Connected.
  4. Click the radio button Catalog Service Domain Id to view the domains available to connect through the Catalog Domain Manager. See the section Using Domain IDs to connect to the data grid for more information.
  5. In the Grid name and Map name fields, enter the names of the target grid and map. The default for the Server servlet application provided is Grid for the grid name and Map1 for the map name. If you are using this client with an XC10 appliance, you must first create a simple grid on the XC10 appliance. Use the name of the simple grid for both the grid name and the map name.
  6. Enter a string value for Key and a string value for Value. Then, select the appropriate CRUD operation; either Insert, Update, Delete, or Get.

Running the GettingStarted sample client

  1. In a command line, navigate to your liberty_installation/bin directory.
  2. To create a server, run the command server.bat create serverName.

    server.bat create gettingstarted

  3. Drop in your GettingStartedWebClient.ear into your liberty_installation/usr/serverName/dropins directory.
  4. You must add the eXtreme Scale web feature, servlets, and jsps to the liberty server's server.xml.
    in the server.xml: 
    <!-- Enable features -->
    
    <featureManager>
    
        <feature>jsp-2.2</feature>
    
        <feature>servlet-3.0</feature>
    
        <feature>eXtremeScale.webapp-1.1</feature>
    
    </featureManager>
  5. To start the server, run the command server.bat start serverName.

    server.bat start gettingstarted

  6. Open a web browser and enter the URL: http://hostname:port/GettingStartedWebClient/ClientServlet

    The hostname is the host name or IP address of Liberty Profile Server and port is the web server port for your Liberty Profile Server. Check the console.log file in liberty_installation/usr/servers/serverName/logs for the URL where the web application is available. See the following example:

    http://localhost:9080/GettingStartedWebClient/ClientServlet/

  7. In the Client Connection section, leave this section blank to use the configured or embedded catalog server, or enter the hostname or IP address and port of the Remote catalog service endpoint in the form of hostname:port.
  8. After entering the catalog service endpoint, click Connect to test the connection. If the connection is successful, you will see true for Client Connected.
  9. Click the radio button Catalog Service Domain Id to view the domains available to connect through the Catalog Domain Manager. See the section Using Domain IDs to connect to the data grid for more information.
  10. In the Grid name and Map name fields, enter the names of the target grid and map. The default for the Server servlet application provided is Grid for the grid name and Map1 for the map name. If you are using this client with an XC10 appliance, you must first create a simple grid on the XC10 appliance. Use the name of the simple grid for both the grid name and the map name.
  11. Enter a string value for Key and a string value for Value. Then, select the appropriate CRUD operation; either Insert, Update, Delete, or Get.

Using Domain IDs to connect to the data grid

In WebSphere eXtreme Scale Version 8.5, you can define a catalog service domain in WebSphere Application Server and programmatically connect it to the grid with the catalog domain manager. Use the following sample code to connect to the data grid if you know the Domain ID of your catalog service domain:

ObjectGridManager ogm = ObjectGridManagerFactory.getObjectGridManager();

CatalogDomainManager cdm = ogm.getCatalogDomainManager();

String endpoints = cdm.getDomainInfo(domainId).getClientCatalogServerEndpoints();

ClientClusterContext ccc = ogm.connect(cep, null, null);

If you do not know the catalog server domain ID in WebSphere Application Server, then you must define one. To define a catalog service domain, complete the following steps:

  1. In a browser, log in to the WebSphere Application Server administrative console.
  2. In the left navigation, expand System administration > WebSphere eXtreme Scale.
  3. Click Catalog service domains, and then click New....
  4. Enter an ID name in the Name field under General Properties. This name is your domain ID.
  5. Select a radio button for either an Existing application server or Remote server.
  6. Enter a value for Catalog server endpoint with the Client, or enter a value for Listener port under Catalog Servers. For more information, see Catalog service domains in the WebSphere eXtreme Scale Information Center.
  7. Click OK.

Importing the samples into Rational Application Developer

The provided project has been tested with Rational Application Developer 7.5.5.3 and later. The following steps guide you through the setup of your eXtreme Scale development environment for applications deployed to a WebSphere Application Server.

  1. Open Rational Application Developer to a new or existing workspace.

    Import the client project

  2. From the File menu, select Import...
  3. Expand the Java EE folder, select EAR file, and click Next.
  4. Browse to the directory of the downloaded sample, select GettingStartWebClient.ear, and then click Open. Select the appropriate target runtime environment, and click Finish.

    You should now see the "client" project in your workspace. If your target runtime environment has WebSphere eXtreme Scale installed, you should not see build errors. However, if there are build errors, then you must resolve them by adding the eXtreme Scale JAR files to the Build Path, as described in the following steps:

  5. Right-click the GettingStartedWebClient project, and select Build Path > Configure Build Path.
  6. Click the Libraries tab, and then click Add External JARS....
  7. Navigate to the <install_root>/lib, and select wsogclient.jar. Click Open and OK.

    The correct classes and libraries are now in the build path, and the project should be error free.

    Import the Server project

  8. From the File menu, select Import....
  9. Expand the Java EE folder, select EAR file, and click Next.
  10. Browse to the directory of the downloaded sample, select GettingStartWebServerEAR.ear, and click Open. Select the appropriate target runtime environment, and click Finish.

    You should now see the "server" project in your workspace. If your target runtime environment has WebSphere eXtreme Scale installed, you should not see build errors. However, if there are build errors,you must resolve them by adding the eXtreme Scale JAR files to the Build Path, as described in the following steps:

  11. Right-click the GettingStartedWebServer project. Then select Build Path > Configure Build Path.
  12. Click the Libraries tab. Then click Add External JARS....
  13. Navigate to the <install_root>/lib, and select wsobjectgrid.jar. Click Open and OK.

    The correct classes and libraries are now in the build path, and the project should be error free.
